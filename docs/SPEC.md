# ぺらぺらアニメメーカー 仕様書 (Draft v0.1)

## 1. 概要

OpenToonzのような本格的な2Dアニメーション制作ソフトウェアをWindows向けに開発する。

- 対象ユーザー: プロ/セミプロのアニメーター、個人クリエイター
- 入力デバイス: ペンタブレット、液晶タブレット、マウス
- 将来的にソフトウェアを公開する予定のため、使用ライブラリは全てフリー/オープンソースライセンスに限定する
- 本格運用に耐える速度・軽さを重視し、C++で開発する

## 2. 技術スタック

| 項目 | 選定 | ライセンス | 備考 |
|---|---|---|---|
| 言語 | C++17/20 | - | パフォーマンス最優先 |
| UIフレームワーク | Qt 6 | LGPLv3(動的リンク) | ドッキングパネル構成が容易。OpenToonzも採用実績あり |
| 描画/レンダリング | OpenGL + glad等のローダー | オープン標準 / MIT | 撮影・合成・エフェクトの拡張を見据える |
| タブレット入力 | Windows Ink API (プライマリ) | Windows SDK標準 | 筆圧・傾き検知 |
| タブレット入力(将来検討) | Wintab | 要個別確認 | 対応時にヘッダのライセンス条項を確認する |
| ビルドシステム | CMake | BSD系 | - |
| 対象OS | Windows 10/11 | - | 将来的なクロスプラットフォーム化の余地は残す |

### ライセンス方針(重要)

- 使用する全ての外部ライブラリはMIT/BSD/Apache-2.0/LGPL/zlib等のフリー/オープンソースライセンスに限定する
- GPL単体ライブラリは将来のクローズド/OSS両対応の選択肢を狭めるため、原則採用しない
- Qtは LGPLv3 + 動的リンクを採用する。これにより以下の義務を満たせば商用・非商用問わず自由に配布できる:
  - QtをLGPL-3.0で利用している旨の明示(取扱説明書等)
  - LGPL-3.0/GPL-3.0のライセンス文書を利用者に提供
  - 著作権表示とライセンス文をUI上に適切に表示
  - 利用者が(修正版の)Qtライブラリに再リンクできる仕組みを提供(動的リンクであれば通常満たされる)
- Qtのモジュールには一部LGPL対象外(GPL専用/商用専用)のものが存在するため、追加モジュールを使う際は都度ライセンスを確認する(未決事項参照)

## 3. アーキテクチャ方針(拡張性)

将来的に以下の機能を追加することを前提に、MVPの段階から拡張を見込んだ設計にする。

- タイムシート(Xsheet)
- タイムライン
- 絵コンテ
- レイアウト
- 撮影(カメラワーク・マルチプレーン合成・エフェクト)

### 設計方針

1. **データモデルを先行して複数対応で設計する**
   Project → Scene → Cut → Cel → Layer → Frame の階層構造を定義する。実際のアニメ制作工程に合わせ、シーンの下にカット(Xsheet/タイムシートを持つ単位)、カットの中にセル(Aセル/Bセル等、タイムシートの列に対応する作画単位)、セルの中にレイヤー(主線/色トレス線/彩色等)、レイヤーの中にフレーム(コマ)を置く。セルの1枚目の絵=各レイヤーのフレーム1の合成。MVPでは最小構成しか使わないが、モデル自体は各階層の複数持ちに対応させておく。

2. **コアとUIを分離する(レイヤードアーキテクチャ)**
   - Core層: データモデル、ファイルI/O、描画エンジン、Undo/Redo
   - UI層: Qtのドッキングウィジェットによるパネル群(キャンバス、オニオンスキン設定、将来のタイムシート/絵コンテ/レイアウトパネルなど)
   - UI層はCore層のAPIを介してのみデータを操作し、新パネルの追加がCore層の変更を必要としないようにする

3. **プロジェクトファイルはバージョン管理可能な形式にする**
   フォーマット(候補: JSON + バイナリ画像データ、またはZIPコンテナ)にスキーマバージョンを持たせ、後方互換を保ちながらタイムシートや絵コンテ等のメタデータを追加できるようにする。

4. **描画エンジンはレイヤー合成を前提にした設計にする**
   MVP時点でもレンダリングパイプラインは「複数レイヤーを合成して1枚の画像にする」処理として実装し、撮影機能(カメラワーク・エフェクト)を後から合成パイプラインに差し込めるようにする。
